# Тезисы (конференция)

- **`tezisy-markdex-konferenciya-2026.docx`** – основной файл для сдачи: поля А4 (верх 5,9 см, низ 6,4 см, слева и справа 4,8 см), Times New Roman 10,5 pt, абзац 1 см, интервал одинарный, выравнивание по ширине. По **требованиям оргкомитета** объём считается **на весь** тезис целиком: от названия до списка литературы, **около 8 тыс. знаков с пробелами** (ориентир 3–5 страниц в заданном оформлении); отдельного лимита «на раздел» в регламенте нет. После `build_tezisy_docx.py` скрипт печатает оценку знаков по абзацам; при необходимости доберите объём или ужмите текст в **`build_tezisy_docx.py`** и пересоберите `.docx`.
- В шапке **нет** плейсхолдеров в квадратных скобках и **нет** угловых скобок вокруг имён тегов в основном тексте; контакты в подписной версии – по требованию оргкомитета.
- **`tezisy-markdex-konferenciya-2026.html`** – предпросмотр; актуальное содержание и оформление задаются скриптом сборки `.docx`.

Пересборка `.docx` после правки скрипта:

```bash
python3 -m venv .venv-tezisy   # один раз
.venv-tezisy/bin/pip install python-docx
.venv-tezisy/bin/python docs/konferenciya/build_tezisy_docx.py
```

Текст и структура задаются в **`build_tezisy_docx.py`** (чтобы не расходиться с Word).

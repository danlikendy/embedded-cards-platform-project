# ЗАКЛЮЧЕНИЕ

В ходе выпускной квалификационной работы достигнута поставленная во введении цель - спроектирован и разработан MVP персональной веб-платформы встроенных карточек внешних источников с MDX-редактором, функцией автодополнения и агрегаторами метаданных, проведена оценка управленческой и технической эффективности решения. Полученные результаты подтверждают исходную гипотезу и закрывают четыре исследовательские задачи, заявленные во введении.

**По первой задаче - анализ теоретических основ и рынка** (разделы 1.1-1.3, 2.1-2.3) - результаты следующие. Систематизированы научные подходы к разработке веб-платформ персонального управления контентом с критическим сопоставлением позиций авторов: Зараменских [3], Васильевой и др. [4, 5], Гриббла и Долматова [13], Михайлова [14]. Установлено, что в существующей научной литературе персональная веб-платформа с интегрированной серверной агрегацией метаданных и MDX-форматом хранения не описана в виде целостной архитектуры. Сравнительный анализ семи ведущих PKM-платформ (Notion, Obsidian, Craft, Roam Research, Bear, Logseq, Typora) по шести релевантным параметрам показал, что ни одна из них не закрывает одновременно требования к нативной поддержке MDX, к переносимой карточке внешнего источника и к независимости от стороннего API агрегации. PESTLE-анализ и матрица RACI зафиксировали внешние ограничения и распределение ответственности при пилотировании; SIPOC-модель и архитектурные диаграммы AS-IS / TO-BE в нотации ArchiMate 3.1 формализовали переход от ручного копирования метаданных к автоматической агрегации. Рыночные оценки получены из открытых обзоров Grand View Research [23], Cnews [24] и TAdviser [25]: глобальный рынок ПО управления знаниями в 2023 г. составил 23,7 млрд долл. (≈ 2,18 трлн руб. по курсу ЦБ РФ на 26.04.2026); CAGR 2024-2030 гг. оценивается на уровне 13-15%. Трассировочная таблица T-01-T-06 и операциональные определения восьми продуктовых метрик M-01-M-08 (табл. 9, 12) задали проверяемые критерии приёмки и измерения. Расчёт TCO по консервативному сценарию показал экономию 6 920 руб./год на пользователе только за счёт сокращения трудозатрат и до 59 840 руб./год - при замене стороннего API собственным агентом агрегации.

**По второй задаче - проектирование архитектуры** (раздел 3.1) - спроектирован контур из четырёх контейнеров в нотации C4, разделённых по типу хранения и жизненному циклу данных: браузерный SPA на React и CodeMirror 6; API Route агрегации на Next.js + node-html-parser; серверный in-memory кэш с TTL 24 ч; хранилище документов в localStorage. Сформулирован формальный контракт API агрегации (DTO с полями url, canonical, title, description, excerpt, wordCount, image, domain, source, cached, fetchedAt) и правило приоритета источников OG → JSON-LD → Twitter Cards → HTML meta → fallback на тело страницы. Дифференцирующее архитектурное решение - явное разделение полей `description` (декларативный OG-текст автора сайта) и `excerpt` (фактический фрагмент тела страницы, извлечённый собственным парсером). Такого разделения нет ни в одной из проанализированных платформ; именно оно составляет ядро предметной новизны работы.

**По третьей задаче - разработка и интеграция компонентов** (разделы 3.2-3.3) - реализованы все компоненты целевого контура: MDX-редактор на базе CodeMirror 6 с debounce-детектором URL 400 мс; компонент UrlSuggestion, отображающий превью карточки до подтверждения вставки и не модифицирующий исходный текст без явного действия пользователя; Quick URL bar для нетехнической аудитории, не требующий понимания JSX-синтаксиса; трёхфазный алгоритм извлечения тела страницы (удаление DOM-шума → поиск семантического контейнера → нормализация и формирование excerpt из 600 символов); компонент LinkCard с отображением реального текста страницы вместо OG-тизера; хуки useMetadata и useDocuments с управлением состояниями жизненного цикла и экспортом .mdx-документа без конвертации формата.

**По четвёртой задаче - тестирование и валидация** (раздел 3.4) - на наборе из 20 URL четырёх категорий все шесть идентификаторов T-01-T-06 закрыты: доля успешных парсингов составила 95% (19/20), медиана времени агрегации - 1 240 мс при NFR ≤ 3 000 мс (запас по производительности 2,4×), P95 - 2 810 мс, доля URL с непустым excerpt ≥ 50 символов - 75%, graceful degradation при сбое сети - 100%, время ответа из кэша - 2-4 мс. UX-оценка по валидированной методике UMUX-Lite [15] на экспертной выборке (n = 7, состав описан в таблице 17) дала средний балл 77,4 - категория «хорошо» (порог 74 балла). Индивидуальные комментарии экспертов подтвердили T-02 (переносимость JSX-компонента) и архитектурное решение о разделении полей description/excerpt как ключевые дифференциаторы продукта.

Дополнительно в работе выполнено экономическое обоснование коммерческой реализуемости проекта (раздел 3.5): расчёт TCO TO-BE с разбивкой по статьям (хостинг, KV-кэш, CDN, домен, амортизация оборудования по линейному методу за 36 мес. согласно НК РФ); оценка применимости 152-ФЗ к пилотной и продакшн-фазам с описанием альтернативных российских облачных платформ (Yandex Cloud, Selectel, VK Cloud); прогнозирование операционных и капитальных затрат на горизонте 3 лет; расчёт NPV (+1 510 тыс. руб. при ставке дисконтирования 28%), PI (11,07), DPP (14-16 мес.) и вспомогательный недисконтированный ROI (159,5%); сценарный анализ с пессимистическим и оптимистическим вариантами. Сформулировано ценностное предложение для трёх клиентских сегментов (раздел 3.6) с количественным сравнением по семи параметрам (табл. 24); продукт демонстрирует TCO 200 руб./год для индивидуального пользователя против 9 600-16 560 руб./год у конкурентов, а для B2B-сегмента - 41 руб./мес/пользователя при цене лицензии 50 000 руб./год за 100 пользователей.

**Научная новизна** работы состоит в трёх взаимосвязанных результатах: (1) предложена и реализована архитектурная интеграция MDX-редактора, автодополнения URL и серверного агрегатора метаданных в единую персональную веб-платформу; (2) обоснована модель хранения карточки внешнего источника как декларативного JSX-компонента с кэшируемым DTO и извлечённым телом страницы; (3) разработана методика сравнительной оценки эффективности такой интеграции, включающая TCO-расчёт, операциональные определения восьми метрик, программу функционального тестирования по идентификаторам требований и UX-оценку UMUX-Lite на экспертной выборке.

**Практическая значимость** результатов выражается в готовом MVP, развёрнутом и валидированном в академической среде, наборе воспроизводимых инженерных решений (контракт API, парсер, схема компонента LinkCard, паттерны хуков), а также в комплекте трассируемых артефактов бизнес-анализа (PESTLE, RACI, SIPOC, ArchiMate AS-IS/TO-BE, T-01-T-06, M-01-M-08), пригодных для тиражирования при создании других персональных инструментов управления знаниями.

**Перспективы развития** проекта формулируются по четырём направлениям. Технологическое: переход с in-memory кэша на распределённый KV-store (Redis / Upstash) для горизонтального масштабирования; внедрение headless-браузера (Puppeteer) для извлечения excerpt с SPA-страниц без SSR; реализация полноценного MDX-компилятора на клиенте; разработка мобильных приложений на React Native. Продуктовое: расширение функциональности (двунаправленные ссылки, граф связей, командная работа, синхронизация между устройствами); интеграция с системами управления библиотечными ссылками (Zotero, Mendeley) для академической аудитории. Регуляторное: при коммерциализации - перенос инфраструктуры на отечественное облако (Yandex Cloud / Selectel) с соблюдением требований 152-ФЗ. Научно-методическое: повторная оценка UX по полной методике SUS на выборке ≥ 30 респондентов с включением нетехнической аудитории; публикация результатов в журнале, индексируемом в РИНЦ; апробация решения на пилотных проектах в Финуниверситете и других вузах-партнёрах.

В совокупности результаты ВКР подтверждают, что проектирование персональной веб-платформы с поддержкой MDX, функцией автодополнения и агрегаторами метаданных является технически реализуемой, экономически обоснованной и востребованной целевой аудиторией задачей; разработанный MVP закрывает идентифицированный разрыв на рынке PKM-инструментов и обладает положительным экономическим эффектом по всем рассчитанным показателям.
